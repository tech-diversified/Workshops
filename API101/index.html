Let's say you have an idea to build an app that lists real-time 911 Fire Department calls (just go with it).

Several ways to get that list:

Data Gathering - Method 1

Buy a police scanner and listen for all 911 calls that go to the Fire Department.



This is time consuming and error prone. AND, you have to constantly listen to the scanner 24/7 to keep it up to date. You'd also have to write them all down or keep a spreadsheet with all of the details.

Data Gathering - Method 2

Have teams of people stationed at every fire station in Seattle. Whenever a fire truck pulls out, follow it and note the details.



Yeah, right.

Data Gathering - Method 3

Let somebody else worry about data collection!
Use an API!



Thankfully, the City of Seattle has a data portal - http://data.seattle.gov


Look at the 911 data: http://data.seattle.gov/resource/kzjm-xkqj.json

Adding the query results in datetime being spit out. http://data.seattle.gov/resource/kzjm-xkqj.json?type=Aid%20Response

Get to know the API

Each API has a specific set of 'endpoints'

Endpoints do a single job - handle a single topic

Find the endpoint(s) that provide the data you want
